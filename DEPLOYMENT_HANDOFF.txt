2026-05-12 Vercel 404 note:

  Symptom:

    Vercel deployment shows Ready, but visiting the assigned domain returns 404: NOT_FOUND.

  Likely cause:

    The project is a monorepo and the deployable SvelteKit app is apps/dashboard.
    If Vercel project Root Directory is the repository root, it can run the turbo build but fail to publish the adapter-vercel output from apps/dashboard as the site root.

  Recommended fix in Vercel settings:

    Root Directory: apps/dashboard
    Framework Preset: SvelteKit
    Install Command: cd ../.. && pnpm install --frozen-lockfile
    Build Command: cd ../.. && pnpm --filter @cio/dashboard build
    Output Directory: leave empty

  After changing settings:

    Redeploy without build cache if possible.

  Current status:

    User is at the step of finding Vercel Deployments/Redeploy after changing settings.
    If the Deployments tab cannot be found, make a small commit and push to main to trigger Vercel automatically.
    Root Directory changes do not affect the already-built deployment until a new deployment runs.
